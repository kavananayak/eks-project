# 🚀 EKS CI/CD Pipeline Project

## 📌 Overview

This project demonstrates a complete **CI/CD pipeline** for deploying a containerized application to AWS EKS using GitHub Actions.

---

## 🏗️ Architecture

```
GitHub → GitHub Actions → Docker → Amazon ECR → Amazon EKS → LoadBalancer → Browser
```

---

## 🔧 Tech Stack

* AWS EKS (Kubernetes)
* AWS ECR (Container Registry)
* GitHub Actions (CI/CD)
* Docker
* kubectl

---

## 🚀 Features

* Automated build and deployment using GitHub Actions
* Docker image creation and push to Amazon ECR
* Kubernetes deployment on Amazon EKS
* LoadBalancer service for external access
* End-to-end CI/CD pipeline

---

## 📁 Project Structure

```
eks-project/
 ├── k8s/
 │    ├── deployment.yaml
 │    ├── service.yaml
 ├── Dockerfile
 ├── index.html
 ├── .github/
 │    └── workflows/
 │         └── deploy.yml
```

---

## ⚙️ Workflow

1. Developer pushes code to GitHub
2. GitHub Actions pipeline is triggered
3. Docker image is built
4. Image is pushed to Amazon ECR
5. Kubernetes manifests are applied to EKS
6. Application is exposed via LoadBalancer

---

## 🌐 Application Access

The application is exposed via AWS LoadBalancer and can be accessed through the external URL generated by Kubernetes service.

---

## 🧠 Key Learnings

* Setting up and managing EKS clusters
* Implementing CI/CD pipelines using GitHub Actions
* Integrating Docker with AWS ECR
* Debugging Kubernetes networking and LoadBalancer issues
* Understanding end-to-end deployment lifecycle
